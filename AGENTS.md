## Introduction

This project, named `rbox`, is a comprehensive toolbox library with C++26 Reflection (p2996).

This project uses CMake to build & run unit tests.

Local configuration (detailed commands to build & run test cases, for example, where to find `libstdc++.so` or `libc++.so`) can be found in `./AGENTS.local.md` (if exists).

## Project Structure

* `./include/rbox`: API & implementation of `rbox` library;
* `./docs`: Documentation of components;
* `./examples`: Examples of usage;
* `./tests`: Unit test cases.

## Reviewing Documents

Please check each document file one-by-one carefully:
* Spelling: whether there exist typos;
* Grammar: whether there exist grammatical mistakes;
* Wording: whether there are unclear or confusing expressions;
* Structure: whether are are bad document structures which harm readability;
* Code correctness: whether there is inconsistency between document and the actual code; whether there is syntax or logic error in the examples.
* Elaboration: whether there can be better wording to help the reader understand.

Output the full reviewing report to ./REVIEW_REPORT.md (or some other file if this file already exists).
